Python包管理安装

发表于 2016-10-29   |  

起因在安装lxml时出现错误，一般使用pip可以解决，由于lxml是cpython的封装，可以视作二进制文件做个wrapper,在install时需要编辑器等支持。

网上搜索了一遍，可以从 (http://www.lfd.uci.edu/~gohlke/pythonlibs/) 使用非官方的Windows二进制的python扩展包，下载过来是wheel格式，也可以使用pip 安装，在安装文件的路径下运行。最先出现的是 .whl is not a supported wheel on this platform 提示。在网站上.whl都有好几个版本，cp27对应于python2.7, cp35对应于python3.5一般容易理解。win32指对应的python的32位版本，而与windows是否32位或64位无关。网上相关出现的错误都在于上述版本问题。我在避开这些坑后，尝试两种版本都仍出现平台不支持的错误安装不成功。

在stackoverflow上发现pip版本过旧也可能导致错误，升级pip版本安装lxml.whl解决：

Python包管理现在事实工业标准是pip无疑，但一些简单老旧的模块会采用随包带setup.py。setup.py是利用distutil,distutil是在2000年发布用于python模块的安装和发布。

easy_install在一些文章指导也有出现，它是setuptool中的安装工具，在2004年发布用于模块的下载、构建、安装和管理。伴随而来的还有eggs文件格式，相当于将支撑文件一起打包发布。
